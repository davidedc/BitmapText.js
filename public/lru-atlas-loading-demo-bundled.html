About this demo

What it shows. Dynamic, on-demand atlas loading with a bounded-size LRU cache. Many text boxes spawn at random positions with random font / style / weight / size and a random short lifetime; the page pulls in atlases as needed, and the LRU evicts and actually unloads old atlases so the resident-atlas count stays bounded — even under N > K stress.

Why it matters. Real apps with large font catalogues can’t hold every atlas in memory. This shows BitmapText.js’s BitmapText.loadFont / BitmapText.unloadAtlas / BitmapText.hasAtlas primitives composing into a working memory-bounded loader, and confirms that drawTextFromAtlas already paints placeholder rectangles when an atlas is missing — so a textbox can render before its atlas exists and self-repaint when it arrives.

How. Each TextBox calls drawTextFromAtlas immediately at construction (placeholders if not yet loaded) and asks the LRU for its primary + smiley-pair atlases. The LRU returns Promise.resolve() if both are warm, otherwise it kicks off the missing loads — evicting the oldest non-pinned entry first if at capacity — and resolves once the loads complete; the textbox then repaints. Atlases-resident in the stats panel reads AtlasDataStore.size() directly, so it physically can’t exceed K. The race where a freshly-loaded atlas could be evicted before its requester paints is defused by a 250 ms pin window on every recent touch and load completion. The FPS histogram (last 120 frames, color-coded by frame budget) makes any reconstruction-induced jank directly visible.
